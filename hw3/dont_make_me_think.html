Web Design Best Practices

Follow convention

Conventions exist because they work well for delivering content in a way that the user will understand. Not only will following them save you time spent re-inventing the wheel, but users will also be comfortable navigating your site because they are already familiar with the conventions as they've seen them on other sites.

Clarity trumps consistency

If you can make something significantly clearer by slightly decreasing the consistency of the site, do it.

Visual hierarchies

Use visual queues to convey meta-information about the content of the site. Related topics should be grouped, either by putting them close together, or even by enclosing them in the same box. Think of a newspaper and how articles, their titles, and related images are visually close together.

Headings

The relative size of elements should also be used to indicate their importance. It is crucial that heading sizes are easy to distinguish with a quick glance at the page.

Bulleted lists

Take advantage of bullet points. These are a great way of creating content that is easy to scan.

  • Visual groupings
  • Distinguishable heading sizes
  • Bulleted lists

Make it obvious what's clickable

Much of a users time is spent searching for the next thing to click, so it's important to make it obvious what's clickable and what isn't. Clickable things should be marked with a different color or style which should also change when hovered over with the mouse.

Mobile
Keep in mind that a large amount of web traffic comes from mobile devices that don't have a mouse, so clickable items need to be identifiable without the ability to hover.

Reduce noise

It is inevitable that web pages will end up with a lot of noise and at some point there may be too many distractions for users to be able to read.

General noise levels
  • Shouting Lots going on, but you can focus on the thing that interests you
  • Disorganization looks like the devs didn't know how to use a grid
  • Clutter just way too much stuff!

Try to stick to shouting. People can pick out the pieces they want without getting too distracted.

Highlight key terms

Scanning a page usually means looking for key words that sum up the purpose of a section of text. You can make this easier for the user by highlighting those key words in a different color, font, or size.

Goodwill

As someone browses your site, the amount that they trust you will change based on their experience. This will affect the likelyhood that they come back and use the site again in the future.

Here are some examples of what can affect a user's goodwill.

Increase
  • Well organized
  • Main tasks are obvious
  • Prices, fees clearly listed
  • Doing extra work so I don't have to
  • Put effort into it
  • Relevant FAQ
  • Printer friendly pages
  • Easy to recover from errors
  • When in doubt, apologize
Decrease
  • Hiding information
  • Not auto-formatting input data
  • Asking for personal info
  • Faux sincerity
  • Excessive marketing
  • Amature looks

When a user enters the site, they may already be at a low level of goodwill because of a bad experience they had elsewhere. Luckily, their goodwill can recharge if if they feel comfortable on your site, and that it answers all of their questions.

Mindless Choices

Information should be grouped in a way that is as unambiguous as possible. This makes choices when clicking through a menu very straight forward.

It may help to pretend that you are playing 20 questions with the user. Regardless of what they are looking for it should fall into one of several non-overlapping categories.

Animal Vegetable Mineral

This classic example holds as long as you assume that everything which is not an animal or vegitable, can be classified as a mineral.

Usability Testing

Do-It-Yourself Testing

Usability testing does not have to be a huge ordeal. It is actually very effective to do frequent and short tests.

Steps
  1. Set up the testing room with a computer and a webcam to show the user's reactions
  2. Set up the observer room with the webcam feed and some snacks
  3. Prompt the user to complete a set of tasks
  4. Evaluate what the most important issues are
Materials
  • 1-3 test subjects
  • Facilitator
  • Prompting Script
  • Webcam
  • Observer room
  • Food to encourage people to show up

Choosing Participants

You might be tempted at first to choose participants who closely represent your users. However, it's better to pick too loosely than too restrictively because it will improve quality over all.

  • Your site will be more widely usable
  • We're all beginners at something
  • You won't insult anyone by making things easy

Deciding What to Fix

It is important to only pick a small number of problems to work on, as changing the design for these problems can affect the workflow of the whole site. Every viewer should write down the three most serious problems that they saw for each participant.

  • Make a list
  • Choose 10 most serious problems
  • Rate them from 1-10
  • Create an ordered list

Designing for Mobile

The biggest difference in designing for a mobile device is the size of the screen. Mobile screens are much smaller and therefore the problem of fighting over screen real-estate is amplified.

Mobile First

The idea of mobile first design is not to worry about the desktop site until the mobile one is done. This way the most important parts of the site are built into it's core.

Typical problems
  • Lot of work
  • Hard to do well

Luckily Twitter already did all of the work and released their css code called Bootstrap to the public. Bootstrap can be used to make responsive mobile first websites and is extremely easy for developers to use. Additionally, it also implements many of the common conventions a user will run into on the web.

Using Bootstrap makes it easier to create any website, even if it is just meant for a desktop site (with the added bonus of it looking nice on mobile too). It mainly consists of predefined CSS classes that developers can add to parts of their page.
